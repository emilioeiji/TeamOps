.\ProductionMonitorProbe.exe ec2-reset-latest

.\ProductionMonitorProbe.exe dashboard


ProductionMonitorProbe.exe [comando] [opções]

COMANDOS DISPONÍVEIS

import
    Executa a importação padrão dos arquivos de produção.
    Importa eventos, atualiza status atuais, executa integração EC2
    Administrator e exibe estatísticas da importação.

    Exemplo:
    ProductionMonitorProbe.exe import


import-profile
    Executa uma importação completa e mede tempos de execução
    para identificar gargalos de performance.

    Opções:
      --date YYYY-MM-DD
      --sector gbareru|dad|ID

    Exemplo:
    ProductionMonitorProbe.exe import-profile --date 2026-06-03 --sector gbareru

    Exibe:
      - tempo total
      - tempo de importação
      - tempo de analytics
      - kadouritsu
      - quantidade de máquinas
      - áreas encontradas


db-index-check
    Verifica se todos os índices necessários do banco SQLite
    existem e estão configurados corretamente.

    Exemplo:
    ProductionMonitorProbe.exe db-index-check


ec2-diagnostics
    Diagnóstico completo da importação EC2 Administrator.

    Exibe:
      - último arquivo importado
      - hash do arquivo
      - quantidade de linhas
      - máquinas carregadas
      - máquinas obsoletas
      - máquinas ignoradas
      - média de processamento
      - amostras importadas
      - amostras descartadas

    Exemplo:
    ProductionMonitorProbe.exe ec2-diagnostics


ec2-reset-latest
    Remove do banco a última importação EC2 Administrator.

    Utilizado para forçar reimportação de um arquivo que já
    foi processado anteriormente.

    Exemplo:
    ProductionMonitorProbe.exe ec2-reset-latest


dashboard
    Gera dashboards de teste utilizando os dados atuais
    do banco de dados.

    Exibe:
      - kadouritsu
      - máquinas rodando
      - máquinas paradas
      - minutos de erro
      - áreas
      - ranking de operadores

    Exemplo:
    ProductionMonitorProbe.exe dashboard


status-hardening
    Executa validações de integridade das classificações
    de status de produção.

    Valida:
      - seeds de status
      - índices
      - regras StopCounts
      - regras StopNoCount
      - cálculo de eficiência

    Exemplo:
    ProductionMonitorProbe.exe status-hardening


status-report
    Gera relatório detalhado dos status encontrados nos eventos.

    Opções:
      --start YYYY-MM-DD
      --end YYYY-MM-DD
      --sector gbareru|dad|ID
      --csv arquivo.csv

    Exemplo:
    ProductionMonitorProbe.exe status-report --sector dad

    Exemplo CSV:
    ProductionMonitorProbe.exe status-report --sector dad --csv relatorio.csv


schema-check
    Verifica a integridade da estrutura do banco sem modificar nada.

    Valida:
      - tabelas
      - colunas
      - índices
      - seeds
      - quick_check do SQLite

    Exemplo:
    ProductionMonitorProbe.exe schema-check


schema-repair
    Corrige automaticamente problemas encontrados na estrutura
    do banco.

    Ações:
      - recria índices ausentes
      - cria colunas faltantes
      - recria seeds obrigatórios

    Exemplo:
    ProductionMonitorProbe.exe schema-repair


production-diagnostics
    Diagnóstico geral da base de produção.

    Exibe:
      - quantidade de eventos
      - status atuais
      - máquinas ativas
      - primeiro evento
      - último evento
      - dashboard resumido

    Exemplo:
    ProductionMonitorProbe.exe production-diagnostics


machine-cleanup
    Localiza máquinas inválidas cadastradas no banco.

    Modos:

      machine-cleanup
          Apenas lista as máquinas suspeitas (DRY RUN)

      machine-cleanup --apply
          Inativa automaticamente as máquinas inválidas

    Exemplo:
    ProductionMonitorProbe.exe machine-cleanup

    Exemplo:
    ProductionMonitorProbe.exe machine-cleanup --apply


demo
    Executa:
      1. import
      2. dashboard

    É o comportamento padrão quando nenhum comando é informado.

    Exemplo:
    ProductionMonitorProbe.exe demo

    Ou simplesmente:
    ProductionMonitorProbe.exe

    ProductionMonitorProbe.exe ec2-diagnostics
