# 管理パネル

## 目的

管理パネルは TeamOps の基本マスタとサポート用情報を管理する画面です。製造、出勤、Haidai、Follow-up、PR/CL、帳票、サポートに影響するため、管理者のみが使用します。

## 画面構成

- 左側メニューで管理対象を選択します。
- 上部に検索やフィルターがあります。
- 一覧で登録済みデータを確認します。
- `Novo`、編集、削除、`Atualizar` などの操作を行います。

## 共通ボタン

- **Novo:** 新しいレコードを作成します。
- **Editar:** 選択行を編集します。
- **Excluir:** 選択行を削除します。確認後に実行されます。
- **Buscar:** 入力した語句で一覧を絞り込みます。
- **Atualizar:** 一覧またはログを再読み込みします。
- **Salvar:** 入力内容を保存します。
- **Cancelar/Fechar:** 変更せずに閉じます。

読み取り専用画面では、`Novo`、編集、削除は無効になります。

## 管理対象

### Turnos

シフトを管理します。オペレーター、出勤、帳票の基準になるため、名称と時間帯を正しく登録します。

### Grupos

グループを管理します。リーダー別確認、帳票フィルター、配置確認で使用されます。

### Setores

セクターを管理します。製造、Haidai、PR/CL、Follow-up、帳票に影響します。

### Locais

ローカル/場所を管理します。設備や製造イベントの集計単位になります。

### Maquinas

設備マスタを管理します。設備コード、ローカル、セクターの不整合は製造モニターや帳票に影響します。

### Equipamentos

Follow-up や運用記録で使う設備/装置を管理します。

### Machine Status

設備ステータスの分類を管理します。稼働、停止、除外、未知ステータスの扱いに影響します。

### Production Display Codes

製造画面で使う表示コードを管理します。色や表示名の不整合は現場確認を難しくします。

### Procedure Times

手順時間を管理します。製造指標や比較に影響する可能性があります。

### Follow-up Reasons / Types

Follow-up の理由と種類を管理します。未登録の場合、記録や分析の品質が下がります。

### Shain

社員/担当者関連の参照マスタです。Sobra de Peca などで使用します。

### Machine Pendencies

設備の未処理/注意情報を確認する読み取り専用領域です。

### System Log

システムログを確認します。日付、ユーザー、モジュール、アクションで絞り込み、障害調査や操作確認に使います。

## 操作時の注意

- マスタ変更は他モジュールへ即時に影響する可能性があります。
- 削除前に、その値が既存データで使われていないか確認してください。
- 製造や出勤に関係するマスタは、現場担当者と確認してから変更してください。
- System Log は調査用であり、通常は編集しません。

## サポート時に確認すること

- どの管理対象を変更したか。
- 変更前後の値。
- 変更したユーザーと時刻。
- 関連するエラーメッセージ。
- 影響が出たモジュール。
