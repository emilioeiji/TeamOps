# ドメイン観察メモ

## 概要

このメモは、コードと既存ドキュメントから読み取れる TeamOps の業務領域を整理したものです。明示されていない業務判断は、運用担当者に確認する必要があります。

## 重要な概念

- `CodigoFJ`: ユーザー、オペレーター、出勤、タスク、帳票をつなぐ重要な識別子。
- Turno: シフト。Dashboard、出勤、製造、タスクに影響。
- Setor/Local/Maquina: 製造と帳票の集計軸。
- Grupo/Lider: リーダー別確認と運用管理の軸。
- Machine Status: Kadouritsu や停止扱いに影響する分類。
- Hikitsugui: 引き継ぎ、読了、返信、添付、修正を管理する情報。
- PR/CL: テンプレートから生成する運用文書。

## 注意点

- オペレーター、ユーザー、シフト、グループの整合性が崩れると、多くの画面に影響します。
- 製造イベントはファイル、設備マスタ、ステータスマスタ、DB 設定に依存します。
- 帳票の正確性は元データの登録品質に依存します。
- 共有フォルダーの権限不足は添付、テンプレート、エクスポート、インポートに影響します。

## 確認が必要な事項

- 現場で正式に使うステータス名と分類。
- PR/CL のテンプレート管理ルール。
- Haidai の公開先と更新頻度。
- 添付ファイルの保管期間。
- ProductionMonitorProbe の実行権限と運用手順。
