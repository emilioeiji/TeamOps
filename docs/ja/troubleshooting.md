# トラブルシューティング

## システムが起動しない

確認:

- 実行ファイルが存在するか。
- 必要 DLL が同じフォルダーにあるか。
- WebView2 Runtime がインストールされているか。
- `App.config` が壊れていないか。
- DB パス、共有フォルダーにアクセスできるか。

## ログインできない

確認:

- ユーザー名とパスワード。
- ユーザーが有効か。
- `CodigoFJ` が正しいか。
- DB が正しい環境を指しているか。

## Dashboard が開かない

原因例:

- ユーザーにオペレーターが紐づいていない。
- オペレーターのシフトが存在しない。
- WebView2 がない。
- DB に必要な seed がない。

## 画面が白い

確認:

- WebView2 Runtime。
- `ui/<module>` の HTML/CSS/JS が配布されているか。
- JS エラー。
- Form 側の初期 payload エラー。

## 帳票が遅い

確認:

- 期間が広すぎないか。
- ネットワーク DB が遅くないか。
- インデックスが不足していないか。
- 対象セクター/グループで絞り込めるか。

製造関連では `ProductionMonitorProbe.exe db-index-check` と `production-diagnostics` を使用します。

## 製造データが表示されない

確認:

- TXT/DAT ファイルの存在。
- BAT の実行結果。
- `DatabasePath`。
- インポート対象日。
- Machine、Local、Status のマスタ。
- 未知ステータス。

## EC2 が更新されない

確認:

- EC2 Administrator のファイル。
- 最新ファイルの日時。
- hash または同一ファイル再取り込みの扱い。
- `ec2-diagnostics` の結果。

## PR/CL が生成されない

確認:

- テンプレートファイルが存在するか。
- 出力フォルダーに書き込み権限があるか。
- ファイル名に使用できない文字がないか。
- Excel テンプレートが壊れていないか。

## 添付ファイルが開けない

確認:

- 添付フォルダー。
- ネットワーク権限。
- ファイルが移動/削除されていないか。
- 同じ DB と添付フォルダーを参照しているか。

## サポートへ連絡するとき

以下を伝えてください。

- ユーザー名。
- モジュール名。
- 発生日時。
- 実行した操作。
- 表示されたエラー全文。
- 対象日、シフト、セクター、オペレーター。
- 可能であれば System Log の関連行。
