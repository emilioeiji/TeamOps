# ユーザーマニュアル

## システム起動

1. TeamOps の実行ファイルを開きます。
2. ログイン画面が表示されるまで待ちます。
3. ユーザー名とパスワードを入力します。
4. ログイン後、メイン Dashboard が表示されます。

システムが起動しない、または DB/フォルダー関連のエラーが出る場合は、表示されたメッセージをサポートへ連絡してください。

## メイン画面

メイン画面は各モジュールへの入口です。表示されるボタンはユーザー権限によって変わる場合があります。

Dashboard で確認できる情報:

- ログインユーザー。
- 紐づくオペレーター。
- シフト。
- 日時。
- 未完了タスク、進行中 Master Card などの概要指標。

主なボタン:

- **Operadores:** オペレーター情報の検索と保守。
- **Atribuicoes:** オペレーターとリーダー/グループの紐づけ。
- **Relatorios:** 運用帳票のハブ。
- **Presenca/Layout:** 出勤状況と配置の確認。
- **Haidai:** TV/モニター表示向けの運用パネル。
- **Follow-up:** 発生事項、理由、対応状況の登録。
- **Tarefas:** タスク管理。
- **Master Card:** 進行中カードの管理。
- **Monitor de Producao:** 設備イベントのインポートと確認。
- **Hikitsugui Criacao:** 引き継ぎ情報の作成。
- **Hikitsugui Leitura Lider:** リーダー向け引き継ぎ確認。
- **Sobra de Peca:** 余剰部品の登録。
- **PR / CL:** PR/CL 文書の作成と参照。
- **Yukyu/Paid Leave:** 有休/休暇関連の確認。
- **Administracao:** 基本マスタと管理情報。
- **Controle de Acesso:** ユーザーと権限の管理。

## モジュール別操作

### Operadores

オペレーターの基本情報、`CodigoFJ`、シフト、セクター/グループを確認します。`CodigoFJ` はログイン、タスク、出勤、ユーザー連携に影響するため、変更前に内容を確認してください。

### Atribuicoes

オペレーター、グループ、リーダーの関係を設定します。設定ミスはグループ別フィルターやリーダー向け確認に影響します。

### Relatorios

帳票ハブでは、Hikitsugui、Follow、Tasks、Operadores、Presenca、Gerencial de Producao、MasterCard、PR、CL、Sobra de Peca を開けます。

Presenca 帳票の基本手順:

1. Dashboard から **Relatorios** を開きます。
2. **Presenca** を選びます。
3. 期間、シフト、セクター、グループ、ステータスを必要に応じて指定します。
4. FJ コードまたは氏名で検索します。
5. 概要カードとオペレーター一覧を確認します。

Gerencial de Producao の基本手順:

1. **Relatorios** を開きます。
2. **Gerencial de Producao** を選びます。
3. 期間、セクター、ローカル、シフト、グループ、オペレーター、設備、Part Code、リーダーで絞り込みます。
4. ランキング、比較、日別傾向、出勤との突合、アラートを確認します。

期間や対象が広い帳票は時間がかかる場合があります。データが空の場合は、まず期間、シフト、セクター、元データのインポート状況を確認してください。

### Presenca/Layout

日付、シフト、セクターを選択し、予定オペレーター、出勤状況、配置を確認します。配置変更は権限があり、正しいレイアウトを確認できる場合のみ行ってください。

### Haidai

運用情報を TV/モニター向けに表示・エクスポートします。モニターが更新されない場合は、エクスポート先フォルダーの権限と画面の再読み込みを確認してください。

### Follow-up

種類、理由、設備/ローカル、説明、担当者、ステータスを登録します。対応が進んだらステータスを更新し、Follow 帳票で集計を確認します。

### Tarefas

タスクには担当者、期限、ステータスを明確に設定します。ステータス変更時は履歴が残るため、進捗に合わせて更新してください。

### Master Card

オペレーター、トレーナー、セクター、設備、説明、日付、ステータスを登録します。Dashboard の件数が合わない場合は、カードのステータスを確認してください。

### Monitor de Producao

TXT/DAT ファイルと設定された BAT を使って製造イベントをインポートします。

基本手順:

1. **Monitor de Producao** を開きます。
2. 日付、セクター、ローカルを確認します。
3. 必要に応じてインポートを実行します。
4. 完了するまで待ちます。
5. 設備、現在ステータス、Kadouritsu、EC2、G-Bareru 予測を確認します。

インポート中に PC を終了しないでください。`ProductionMonitorProbe` のデータ変更コマンドは技術チームが実行します。

### Hikitsugui

引き継ぎ情報の作成、リーダー確認、オペレーター確認を行います。添付ファイルが開けない場合は、添付フォルダーの権限と接続を確認してください。

### Sobra de Peca

ロット、オペレーター、設備、品目、重量、数量、shain、リーダー、備考を登録します。後で検索しやすいよう、品目名と備考は標準化してください。

### PR と CL

セクター、カテゴリ、優先度、タイトル、ファイル名を入力し、Excel テンプレートから文書を生成します。テンプレートや保存先フォルダーがない場合はサポートへ連絡してください。

### Yukyu/Paid Leave

有休、休暇、Todoke、理由、確認状況を管理します。オペレーター、シフト、理由の登録が不足していると結果が不整合になります。

### Administracao

シフト、グループ、セクター、ローカル、設備、Machine Status、製造表示コード、手順時間、Follow-up マスタ、shain、Machine Pendencies、System Log を管理します。詳細は `docs/admin-panel.md` を参照してください。

### Controle de Acesso

ユーザー作成/編集、アクセスレベル、`CodigoFJ`、パスワード変更、ユーザー有効/無効を管理します。パスワードは BCrypt で保護され、元の値は確認できません。

### OperatorApp

オペレーターが Hikitsugui を確認する別アプリです。TeamOps 本体と同じ運用 DB を参照する必要があります。

## よくある運用

- シフト開始時にログインします。
- 氏名、シフト、セクターが正しいか確認します。
- 必要なモジュールを Dashboard から開きます。
- 日付、シフト、セクター、オペレーター、添付を正しく入力します。
- 画面を閉じる前に保存します。
- 帳票では印刷/エクスポート前にフィルターを確認します。

## エラー時

1. メッセージを記録します。
2. 可能であればエラー画面だけを閉じます。
3. ネットワークと共有フォルダーを確認します。
4. もう一度モジュールを開きます。
5. 解決しない場合は、ユーザー、モジュール、時刻、操作内容をサポートへ連絡します。
